# Архитектура AppСтрой

Код переводится на модульную структуру постепенно, без переписывания приложения и без поломки старых импортов.

## Основные папки

- `lib/app` — тема и настройки всего приложения.
- `lib/core` — общие ошибки, форматирование и технические сервисы.
- `lib/features` — функциональные модули: авторизация, сотрудники, табель, выплаты, задачи и объекты.
- `lib/shared` — переиспользуемые виджеты.
- `lib/data`, `lib/models`, `lib/screens` — старые пути. На время перехода они могут содержать совместимые `export`, чтобы существующий код продолжал работать.

## Правила

1. Экран отвечает за состояние и навигацию, а крупные части интерфейса выносятся в отдельные виджеты.
2. Запросы к Supabase находятся в репозиториях, а не внутри UI.
3. Цвета и стандартные стили задаются только через `AppTheme` и `AppColors`.
4. Текст ошибок очищается централизованно через `ErrorText`.
5. Названия объектов и пользователей нельзя жёстко прописывать в коде.
6. Перенос старых файлов выполняется по одному модулю с сохранением совместимости.

## Уже перенесено

- тема приложения;
- модель профиля пользователя;
- репозиторий пользователя;
- экран импорта личных данных сотрудников;
- общий виджет сообщения об успехе или ошибке.

Следующими модулями для переноса являются сотрудники, выплаты, табель и задачи.
