# Деплой AppСтрой

## Источники истины

- исходный код: `13off/skbs_app`;
- production backend: Supabase-проект `skbs`;
- опубликованная web-сборка хранится отдельно от исходников;
- мобильный релиз отмечается в `release/mobile-release.txt`.

Секреты и `service_role` не хранятся в репозитории.

## Pull Request

Перед слиянием PR должен пройти:

```bash
flutter pub get --enforce-lockfile
flutter analyze --no-fatal-infos --no-fatal-warnings
flutter test --reporter compact
flutter build web --release --base-href /appstroy-web/
```

Изменённые Edge Functions дополнительно проходят `deno check`. SQL-файлы проверяются на пустое содержимое и conflict markers; содержательные миграции отдельно компилируются и проверяются в Supabase.

## Миграции базы

1. Изучить действующие таблицы, функции, grants и RLS.
2. Проверить SQL в транзакции с `ROLLBACK` или на development branch.
3. Выполнить контрольные запросы до изменения.
4. Применить миграцию один раз с понятным именем.
5. Повторить контрольные запросы под нужными ролями.
6. Запустить Security и Performance Advisors.
7. Зафиксировать миграцию в репозитории и PR.

Не применять несовместимое удаление колонок, пока используются старые клиенты.

## Edge Functions

- деплоить полный каталог функции, включая относительные зависимости;
- пользовательские функции по умолчанию разворачивать с `verify_jwt=true`;
- после deploy проверить `ACTIVE`, номер версии и настройку JWT;
- выполнить негативную проверку без токена;
- проверить разрешённую и запрещённую роль;
- просмотреть edge logs после первого вызова.

## Web/PWA

Production-сборка:

```bash
flutter build web --release --base-href /appstroy-web/
```

После публикации проверить:

- вход и восстановление сессии;
- переключение вкладок;
- выбранный объект;
- сотрудников, табель, задачи и выплаты;
- обновление service worker;
- установку PWA;
- скачивание Excel, DOCX и ZIP.

## Android и iOS

Мобильная сборка запускается только при осознанном изменении маркера мобильного релиза. PWA-изменения не должны автоматически выпускать новый APK/IPA.

## macOS Intel

Workflow `Build macOS 11 Intel` запускается вручную. Он:

- использует Flutter из `.fvmrc`;
- генерирует платформенный проект;
- устанавливает минимальную macOS 11.0;
- запускает анализатор и тесты;
- собирает Intel release;
- выполняет ad-hoc подпись;
- прикладывает ZIP, DMG и SHA-256.

Это внутренняя сборка. Для публичного распространения потребуются Developer ID, hardened runtime и notarization.

## Откат

- Flutter/PWA: повторно опубликовать последнюю проверенную сборку;
- Edge Function: развернуть предыдущий проверенный комплект файлов;
- база: предпочитать совместимую исправляющую миграцию, а не ручное удаление истории;
- данные: перед необратимой операцией сохранять контрольный список и manifest затрагиваемых сущностей.
