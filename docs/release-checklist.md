# Чек-лист релиза

## До PR

- [ ] Изменение ограничено заявленной задачей.
- [ ] Рабочий процесс, роли и дизайн не меняются случайно.
- [ ] Нет секретов и реальных персональных данных.
- [ ] Добавлены unit-, regression- или contract-тесты.
- [ ] Новые записи учитывают компанию и объект.
- [ ] Необратимые действия требуют отдельного подтверждения.

## Код Flutter

- [ ] `flutter pub get --enforce-lockfile`
- [ ] `flutter analyze --no-fatal-infos --no-fatal-warnings`
- [ ] `flutter test --reporter compact`
- [ ] `flutter build web --release --base-href /appstroy-web/`
- [ ] Проверены ошибки загрузки и повторная попытка.
- [ ] Проверена работа на узком экране.
- [ ] Скачивание файлов работает на целевых платформах.

## База и RLS

- [ ] SQL проверен в транзакции или development branch.
- [ ] Выполнены контрольные запросы до и после.
- [ ] Новые таблицы в exposed schema имеют RLS.
- [ ] `UPDATE` имеет `USING` и `WITH CHECK`.
- [ ] `SECURITY DEFINER` имеет фиксированный `search_path`.
- [ ] Функция проверяет `auth.uid()`, компанию и роль.
- [ ] Лишний `EXECUTE` для `anon` и `authenticated` отозван.
- [ ] Запущены Security и Performance Advisors.

## Edge Functions

- [ ] `deno check` пройден.
- [ ] Все относительные файлы включены в deploy.
- [ ] `verify_jwt=true`, если функция не является проверенным webhook/job.
- [ ] Нет `service_role` в read-only функции пользователя.
- [ ] Без JWT функция отказывает.
- [ ] Разрешённая роль работает.
- [ ] Запрещённая роль получает отказ.
- [ ] Production-версия имеет статус `ACTIVE`.

## ИИ-действия

- [ ] Сервер только предлагает действие.
- [ ] Критичные поля видны до подтверждения.
- [ ] Действие проходит единый координатор.
- [ ] Исходное предложение сохраняется в аудите.
- [ ] Отмена и ошибка фиксируются корректно.
- [ ] Персональные документы и реквизиты не отправляются ИИ.

## Документы кандидата

- [ ] Подтверждено согласие на обработку данных.
- [ ] Файлы повторно проверены через RLS.
- [ ] Тестовые копии исключены из production-пакета.
- [ ] Нет Storage-путей в ответе ИИ.
- [ ] Имена файлов очищены.
- [ ] Ограничения размера применены до и после скачивания.
- [ ] Манифест предупреждает о неподписанных документах.
- [ ] Неизвестные реквизиты не выдумываются.
- [ ] Согласие и трудовой договор утверждены ответственными лицами до подписания.

## Gate реальных персональных документов

Этот раздел обязателен только для релиза, который включает промышленную работу с реальными паспортами, СНИЛС, ИНН, банковскими или иными закрытыми документами.

- [ ] `config/personal-data-production-gate.json` имеет `real_personal_documents_enabled: true`.
- [ ] Все значения `evidence` равны `true`.
- [ ] Заполнены `approved_by` и `approved_at`.
- [ ] Подтверждено российское расположение основной базы, Storage, резервных копий и журналов.
- [ ] Утверждены реквизиты оператора и форма согласия.
- [ ] Утверждены сроки хранения, архивирования и удаления.
- [ ] Проверен журнал открытия, скачивания, изменения и удаления.
- [ ] Проведён успешный backup/restore test.
- [ ] Проверено отключение доступа при увольнении и смене роли.
- [ ] Назначен ответственный за инциденты.
- [ ] Приложены проверяемые доказательства и письменное решение о запуске.

Если хотя бы один пункт не выполнен, production-режим реальных документов остаётся выключенным; допускаются только тестовые, обезличенные или специально маркированные копии.

## После merge

- [ ] Merge выполнен с ожидаемым SHA.
- [ ] Нужные миграции применены.
- [ ] Нужные Edge Functions развёрнуты.
- [ ] PWA опубликована и открывается.
- [ ] Выполнен smoke-тест критичных маршрутов.
- [ ] Проверены production logs.
- [ ] Маркер мобильного релиза менялся только осознанно.
- [ ] Зафиксирована точка отката.
