# Тестовые учётные записи для ролевой приёмки

Этот файл не содержит паролей, адресов электронной почты или production-идентификаторов.

## Обязательный набор

Для полной live-приёмки нужны отдельные активные тестовые членства одной тестовой компании:

- администратор;
- разработчик;
- прораб с назначенным тестовым объектом;
- HR-менеджер;
- бухгалтер;
- юрист.

## Требования

- компания должна быть отдельной от рабочей ООО «СКБС»;
- ФИО, телефоны, документы, выплаты и объекты — только вымышленные;
- учётные записи не должны иметь дополнительные роли сверх проверяемой;
- прорабу назначается ровно один тестовый объект;
- real personal-data gate остаётся закрытым;
- после приёмки временные учётные записи отключаются либо остаются только в тестовой компании.

## Порядок проверки

1. Войти тестовой учётной записью нужной профессии.
2. Открыть **Система → Ролевая приёмка** либо соответствующий профессиональный раздел.
3. Выбрать ту же профессию.
4. Убедиться, что статус «Live-приёмка» активен.
5. Проверить отсутствие красных ошибок.
6. Для прораба убедиться, что указан правильный тестовый объект.
7. Зафиксировать результат и выйти из учётной записи.

Клиентский preview администратора или разработчика не заменяет этот порядок: preview не меняет JWT и серверную роль.
