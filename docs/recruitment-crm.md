# CRM кандидатов AppСтрой

CRM не создаёт вторую базу кандидатов. Она является новым представлением существующего кадрового контура:

- `recruitment_applications` — карточки кандидатов;
- `recruitment_status_history` — история переходов;
- существующие документы, Telegram-диалоги, оформление, отправка на объект и архив остаются связанными с той же заявкой.

## Канбан

Колонки соответствуют кадровым этапам:

1. Новые;
2. Ждём документы;
3. Косяки;
4. Готовы;
5. Билеты;
6. Завершено;
7. Резерв;
8. Отказ.

Перетаскивание карточки меняет реальный статус через `RecruitmentRepository.updateStatus`. На web используется обычное перетаскивание мышью, на мобильном устройстве — удержание.

## Представления и фильтры

- Канбан и список используют одну выборку данных.
- Поиск работает по ФИО, телефону, вакансии, объекту, гражданству, опыту, комментарию и статусу.
- Доступны фильтры по объекту и вакансии.
- Список дополнительно фильтруется по этапам.
- Сводка показывает общее количество, активную работу, готовых к отправке и кандидатов, требующих внимания.

## Действия

Из карточки доступны:

- открытие существующего подробного профиля;
- переход на другой этап;
- архивирование;
- переход к документам, переписке и оформлению через существующий экран кандидата.
